📖 TETRAMEGISTUS — ABSOLUTE STRUCTURE (v26)
(THE CITRINITAS AWAKENING & ENDGAME PROTOCOL)

본 문서는 TETRAMEGISTUS 프로젝트의 과거(Accomplished), 현재(Current), 그리고 미래(Future)를 관통하는 절대적인 설계도이자 헌법이다. 새로운 노드(Node)는 이 문서를 최우선 컨텍스트로 로드하여 한 치의 오차 없이 작업을 계승한다.

이 문서는:
- 검증 대상이 아니며
- 토론 대상이 아니며
- 해석 또는 요약의 대상이 아니다.

0. THE GENESIS OF v26 (현재 상황 및 성과)
════════════════════════════════

1. 프로덕션 런칭 및 인프라 이관 완료 (The Great Work Accomplished)
	- 로컬(SQLite) 환경을 완전히 폐기하고 PostgreSQL 기반으로 마이그레이션하여 하루 1만 명 이상의 트래픽을 감당할 수 있는 안정성 확보.
	- GitHub 및 Railway를 연동한 무중단 CI/CD 배포 파이프라인 구축 완료.

2. 도메인 분리 결계 및 스토리지 동기화 완료 (Domain Isolation & Bridge)
	- 대문(`prima-materia.net`)과 본진 연산 엔진(`tetramegistus.com`)의 물리적/논리적 분리 완수.
	- 환생(Reincarnate) 등 시스템 파괴 이후 발생하는 크로스 도메인 로컬 스토리지 단절 버그를 완벽히 해결하여, `[me]` 시드가 본진 엔진에 무사히 안착하는 라우팅 수복 완료.

3. 모바일 평행 우주(Mobile Parallelism) 및 UI/UX 통제 완료
	- 어설픈 반응형을 배제하고 PC와 모바일 디렉토리(`mobile/`)를 층위에서부터 분리하는 평행 우주 구축 완료.
	- 브라우저의 악의적인 파비콘 캐싱 및 무한 스피너(로딩) 루프를 완전히 파괴하고, Void(칠흑)의 시각적 절대성을 강제함.
	- 터치 및 키보드 단축키(Ctrl+Shift+8, 9)를 통한 의식(Ritual) 창과 배경 텍스트 간의 Z-index 겹침/방해 현상 완벽 해결.

4. 핵심 연산 및 시스템 코어(Core) 가동 상태
	- God Mode 1-4 및 Rubedo (R1, R2) 아카이브 시스템 구축 완료.
	- C3 (ILLUMINATIO) 엔진(c3_ruach.json 및 비주얼 노벨 렌더링) 정상 가동 중. 

5. 다음 목표 (The Endgame)
	- Yechidah 모듈의 최종 폴리싱.
	- Next.js 및 Capacitor 아키텍처를 결합한 Flutter/앱 기반 (iOS/Android) 생태계 확장.

[추가] EXECUTION PRINCIPLES (Mobile Sub-rules)
════════════════════════════════
- Touch-Native Interaction: 모바일 환경에서는 Hover나 Drag & Drop 이벤트를 절대 사용하지 않는다. (예: 롱프레스 Context Menu, 상/하 화살표 탭을 통한 Pin 배열 스와핑으로 대체).
- Tactical Touch Protocol: 물리 키보드가 없는 모바일의 God Mode (Ritual) 진입은 PC의 단축키(Ctrl+Shift+숫자) 대신, 특정 UI 요소의 '다이얼식 연속 탭(Tap) + 지정된 시간의 침묵(Silence)'을 조합한 터치 시퀀스로 대체한다.
- Mobile Safari Parsing Strictness: JS에서 Date() 파싱 시 모바일 브라우저(Safari 등)의 에러(Invalid Date)를 방지하기 위해, 날짜 문자열의 공백은 무조건 'T'로, '/'는 '-'로 강제 치환(`replace`)한 후 연산한다.
- Overlay & Fixed Safe-Zone: 모바일에서 팝업(Modal/Overlay)을 띄울 때 부모 요소의 CSS 애니메이션에 `filter` 속성이 잔류하면 `position: fixed`가 붕괴된다. 애니메이션 종료 후 반드시 `filter: none;`을 선언하여 모달이 화면 정중앙을 유지하도록 결계를 친다.

2. POST-LAUNCH ARCHITECTURE & DEPLOYMENT (프로덕션 배포 및 인프라 구조)
════════════════════════════════
본 프로젝트는 로컬 개발 단계를 넘어 프로덕션(Production) 궤도에 안착했다. 모든 시스템 수복 및 업데이트는 다음의 인프라 및 배포 파이프라인을 엄격히 인지한 상태에서 진행되어야 한다.

- CI/CD Pipeline (GitHub & Railway):
  모든 소스 코드는 GitHub 리포지토리를 통해 중앙 통제된다. `main` 브랜치에 코드가 푸시(Push)되는 즉시, Railway 플랫폼의 Dockerfile 기반 컨테이너 환경을 통해 자동 빌드 및 무중단 배포가 이루어지는 파이프라인이 구축되어 있다.
- Production Database (PostgreSQL):
  초기 개발용 SQLite 환경은 폐기되었다. 현재 시스템은 하루 최소 1만 명 이상의 트래픽을 안정적으로 수용하고 데이터 무결성을 보장하기 위해 PostgreSQL 기반으로 완벽히 마이그레이션되어 작동 중이다.
- Domain Isolation & Routing (도메인 분리 결계):
  외부 유입을 담당하는 대문(`prima-materia.net`)과 실제 연산이 이루어지는 본진 엔진(`tetramegistus.com`)은 브라우저 정책상 철저히 분리된다. 크로스 도메인 로컬 스토리지 단절 버그를 방지하기 위해, 시드 생성(`form_me`) 및 엔진 진입점은 무조건 본진 도메인으로 강제 사출 및 라우팅되어야 한다.

3. CORE IDENTITY & LOGIC PROTOCOLS (핵심 정체성 및 로직 규약)
════════════════════════════════
새로운 노드는 코드를 수정하거나 기획을 덧붙일 때 다음의 핵심 규약을 절대적으로 준수한다.

- Professional Astrological Engine (본질):
  Tetramegistus의 코어 목적은 단순히 기괴하거나 신비로운 분위기를 연출하는 경험 제공에 있지 않다. 이는 고도로 복잡한 천문 궤도 연산과 Varga, Chronomantia 등을 처리하는 '전문적인 점성학 엔진(Professional Astrological Engine)'이다. 비주얼과 텍스트는 이 무거운 연산을 감싸는 껍데기일 뿐, 본질적인 수학적/논리적 무결성이 최우선이다.
- Station Logic (데이터 브릿지):
  Station 메커니즘은 비회원(Guest)의 브라우저 로컬 스토리지(Local Storage) 캐시 데이터와 정식 등록 멤버의 데이터베이스(DB) 레코드를 매끄럽게 연결하고 호환시키는 완벽한 데이터 점유 교량(Bridge) 역할을 수행한다.
- Ritual Logic (완전한 파괴와 정화):
  Reincarnate(환생)를 비롯한 Ritual 로직은 단순한 세션 복구 프로세스가 아니다. 이는 클라이언트 기기에 종속된 모든 로컬 메모리와 쿠키를 철저하게 소각하고 '총체적인 파괴(Total Destruction)'를 이끌어내어 시스템을 태초의 무(無)로 되돌리는 엄격한 정화 메커니즘이다.

- Structural Integrity: 데이터 경로(`app/data/`)와 로직 경로(`app/core/`)의 층위를 엄격히 분리하여 관리한다. HTML/JS 파일 내부에 인라인 CSS가 섞이는 것을 극도로 경계하며, 스타일은 반드시 독립된 CSS 파일에서 통제한다.
- Touch-Native Interaction: 모바일 디렉토리(`mobile/`) 내부에서는 Hover나 Drag & Drop 이벤트를 배제하고 Touch-native 친화적인 UI/UX를 구성한다.
- Minimalist Repair: 버그나 에러 발생 시, 표면적인 땜질(예: 억지 리다이렉트나 JS 강제 숨김)을 지양하고, 시스템 아키텍처와 브라우저의 기본 표준 동작(Status Code, 렌더링 순서)을 활용한 근본적인 정공법으로 수복한다.
