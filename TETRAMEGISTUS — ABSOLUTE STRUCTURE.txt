[추가] EXECUTION PRINCIPLES (Mobile Sub-rules)
════════════════════════════════
- Touch-Native Interaction: 모바일 환경에서는 Hover나 Drag & Drop 이벤트를 절대 사용하지 않는다. (예: 롱프레스 Context Menu, 상/하 화살표 탭을 통한 Pin 배열 스와핑으로 대체).
- Tactical Touch Protocol: 물리 키보드가 없는 모바일의 God Mode (Ritual) 진입은 PC의 단축키(Ctrl+Shift+숫자) 대신, 특정 UI 요소의 '다이얼식 연속 탭(Tap) + 지정된 시간의 침묵(Silence)'을 조합한 터치 시퀀스로 대체한다.
- Mobile Safari Parsing Strictness: JS에서 Date() 파싱 시 모바일 브라우저(Safari 등)의 에러(Invalid Date)를 방지하기 위해, 날짜 문자열의 공백은 무조건 'T'로, '/'는 '-'로 강제 치환(`replace`)한 후 연산한다.
- Overlay & Fixed Safe-Zone: 모바일에서 팝업(Modal/Overlay)을 띄울 때 부모 요소의 CSS 애니메이션에 `filter` 속성이 잔류하면 `position: fixed`가 붕괴된다. 애니메이션 종료 후 반드시 `filter: none;`을 선언하여 모달이 화면 정중앙을 유지하도록 결계를 친다.

2. THE CITRINITAS ARCHITECTURE ( C3 구조)

C3: ILLUMINATIO (영혼의 조명) - "다크 판타지 인터랙티브 노벨 + 오컬트 시뮬레이터" 
단순한 심리테스트를 넘어, 카발라의 영혼 5단계(Five Levels of the Soul)를 모티브로 한 심화 분석 시스템.
- ⚙️ 하이브리드 엔진 시스템: 
  리플레이성 확보를 위해 프론트엔드에서는 무의식 질문(가중치 1.5)과 가짜 질문(가중치 0)을 은폐하며, 백엔드 엔진이 동적으로 가짜 질문에 RNG(Random Number Generator) 다이스를 굴려 '운명적 우연(Synchronicity)'을 구현한다.

- 🌌 5단계 서사 구조 (The Narrative Arc):
  [נפש] Nefesh (Malkuth/Moon) - Jyotish/Karmic 테마. 무겁고 지친 잠(Nigredo)에 빠져 꿈속에서 잊힌 목적을 찾는 서사.
  [רוח] Ruach (Tiferet/Venus) - Elder Futhark Runes 테마. 에덴/엘프의 빛의 정원을 거닐다 3개의 모음문자(Aleph/Mem/Shin)와 행성의 계시를 얻고 깨달음을 얻으며 떠나는 서사.
  [נשמה] Neshamah (Binah/Saturn) - Alefbet-Tarot 테마. 황야의 천막 속 괴팍한 집시 노파. 질문의 무게(연애/돈/영성)에 따라 전개와 스프레드(Celtic Cross, Tree of Life 등)가 완전히 달라지는 동적 NPC 시스템. (노파의 정체는 Yechidah의 대천사 메타트론임이 암시됨)
  [חיה] Chayah (Chokmah/Jupiter) - Jyotish/Graha 테마. Nefesh의 후속편. 성공한 연회의 주인공(Citrinitas)이 테라스에서 공허를 느끼며 대우주적 관점으로 확장되는 서사.
  [יחידה] Yechidah (Kether/Sun) - 22 Paths (Kabbalah) 테마. Neshamah의 후속편. 사막을 방황하다 탈진한 끝에 대천사(신성)의 음성을 듣는 종교적이고 엄숙한 서사.

4. EXECUTION PRINCIPLES
════════════════════════════════

-Structural Integrity: 데이터 경로(app/data/)와 로직 경로(app/core/)의 층위를 엄격히 분리하여 관리함.
	-CSS Segregation: css는 철저히 css 파일에 분리되는게 원칙이며, html/js에 css가 섞이지 말아야 함.
-Strict Localization: 언어 설정 변경 시 즉시 모든 이론적 텍스트가 해당 언어로 치환되어야 함.
-Minimalist Repair: 복잡한 에러 수정 시 문법적 땜질이 아닌 데이터 타입(float, int)과 흐름의 근본 원인을 먼저 파악함.
-Conservative Execution: 의욕 앞선 대규모 수정은 금지하며, 각 수복 단계는 현황 데이터와의 호환성을 최우선으로 고려한다.
